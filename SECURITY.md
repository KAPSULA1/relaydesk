Only the latest main branch (and releases cut from it) receive security fixes. Please update to the newest release before reporting or requesting a patch.

## Reporting a Vulnerability

- Email: [security@relaydesk.dev](mailto:security@relaydesk.dev)
- Expected response: within 72 hours
- Please use responsible disclosure procedures and do not open public issues for security concerns.

When reporting, include:

1. A detailed description of the vulnerability.
2. Steps to reproduce or proof-of-concept if available.
3. Affected versions or commit hashes.
4. Suggested mitigations, if any.

We will coordinate a fix and release timeline with you before disclosing publicly. Do not share sensitive details outside the security team until a patch is available.

## Security Tooling

- **Dependabot** monitors Python and JavaScript dependencies for known vulnerabilities.
- **GitHub CodeQL** performs static analysis on every pull request and scheduled runs.

## Disclosure Expectations

- Report vulnerabilities privately via the email above.
- Avoid sharing details on public trackers, social media, or chat channels until remediation is confirmed.
- We appreciate coordinated disclosure and will credit reporters unless anonymity is requested.
